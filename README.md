渠道系统 MCP 查询 Skill。

用于在 AI Agent 中查询渠道系统相关数据，包括：
- 个人汇总
- 产品汇总
- 供应商 / 供应商账号
- 子渠道 / 分配单
- 结算列表 / 结算详情 / 结算评分
- 推广明细（按天 / 实时）
- 结算支出 / 财务打款列表

> 当前版本仅支持**查询类能力**，不包含创建、修改、删除、审批等写操作。

---

## 一、安装 MCP 服务

请先在你的 AI 客户端中添加渠道系统 MCP 服务。

把配置复制到你的 MCP 配置页面中，然后安装 MCP 服务。

---

## 二、安装 Skill

在安装完 MCP 服务后，再安装本 Skill。

### 安装SKILL

将本仓库中的 `channel-query-skill` 目录放入你的 OpenClaw 工作区 `skills/` 目录下。

示例目录结构：

```text
<workspace>/skills/channel-query-skill/
├── SKILL.md
└── references/
    ├── usage-guide.md
    ├── tool-mapping.md
    ├── field-notes.md
    ├── output-templates.md
    ├── examples.md
    └── test-cases.md
```

---

## 三、开始使用

安装完成后，你就可以直接问你的 AI Agent。

例如：

- 帮我查 2026-04-01 到 2026-04-07 某供应商的个人汇总
- 帮我查产品“抖漫”2 月第一周的结算列表
- 帮我查 settlementNo=JS20260203225304378 的评分情况
- 帮我查 4 月份已支付的财务打款记录

---

## Skill 能做什么

当前支持的核心查询能力：

### 汇总类
- 查询个人汇总
- 查询产品汇总

### 基础信息类
- 查询供应商
- 查询供应商账号
- 查询子渠道
- 查询产品 / 马甲包
- 查询决策负责人

### 业务单据类
- 查询分配单
- 查询结算列表
- 查询结算详情
- 查询结算评分
- 查询推广明细（按天 / 实时）

### 财务类
- 查询结算支出
- 查询财务打款列表

## 使用规则

使用本 Skill 时，请遵守以下规则：

1. 先做时间归一化，再做名称转 ID，再做正式查询。
2. 正式查询如果依赖 ID，不要直接把中文名传给下游查询接口。
3. 同名、多候选、歧义命中时，必须先澄清，不能默认取第一条。
4. 所有 tool 调用参数都必须包在最外层 `request` 对象里。
5. `query_toll_and_free_score` 已停用，不再使用。
6. 敏感字段默认不直接输出，包括账号、密码、登录地址、登录 IP、收款地址等；必要时应先脱敏。

---

## 推荐提问方式

为了提高命中率，建议直接给出：
- 明确时间范围
- 供应商名 / 产品名 / settlementNo
- 如果已知，也可以直接给 ID

推荐示例：

- 帮我查供应商“XX”在 2026-04-01 到 2026-04-07 的个人汇总
- 帮我查产品“抖漫”在 2026-02-01 到 2026-02-07 的结算列表
- 帮我查 settlementNo=JS20260203225304378 的评分明细
- 帮我查 2026-04-01 到 2026-04-30 的已支付财务打款记录
